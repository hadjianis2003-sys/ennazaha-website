# Data Models: Headless CMS (Strapi/Sanity)

**Context**: Core Website MVP (`001-1`)

## 1. Project (Residence)
Represents a physical real estate development.

**Fields**:
- `name` (String, Required) - e.g., "RĂ©sidence Les Jasmins"
- `slug` (String, Required, Unique) - e.g., "residence-les-jasmins"
- `city` (String, Required) - e.g., "Draria"
- `status` (Enum: Ready, Ongoing, New)
- `price_from` (Number, Required)
- `delivery_date` (Date, Optional)
- `description_ar` (Rich Text, Required)
- `description_fr` (Rich Text, Required)
- `map_coords` (JSON - `{lat, lng}`, Required)
- `cover_image` (Media, Required)
- `gallery` (Media Array, Required)
- `brochure_pdf` (Media, Optional)

## 2. Apartment Unit
Belongs to a Project. Used in the Project Detail specs table.

**Fields**:
- `project` (Relation to Project, Required)
- `type` (Enum: F2, F3, F4, Commercial, Required)
- `surface_min` (Number, square meters)
- `surface_max` (Number, square meters)
- `price` (Number, Optional)
- `status` (Enum: Available, Reserved, Sold)

## 3. Testimonial
Reviews from existing buyers displayed on the homepage slider.

**Fields**:
- `customer_name_ar` (String)
- `customer_name_fr` (String)
- `residence_name` (String)
- `rating` (Number, 1-5)
- `content_ar` (Text, Required)
- `content_fr` (Text, Required)

## 4. Lead (CRM Mapping)
Standard structure for data generated by the frontend forms.

**Fields (Contact / Project Inquiry)**:
- `fullName` (String, Required)
- `email` (Email, Required)
- `phone` (String, Required)
- `projectContext` (String, Optional) - Filled automatically if submitted from a project page.

**Fields (Waitlist)**:
- `fullName` (String, Required)
- `email` (Email, Required)
- `phone` (String, Required)
- `preferredCity` (String, Required)
